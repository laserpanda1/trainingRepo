Несмотря на схожесть, подход с именованными запросами вводит сведения о персистентности с заметками @NamedQuery,
 что не всегда считается хорошей практикой. Внимательный читатель может возразить, что подход @Query также указывает
 собственные SQL-запросы внутри класса Java, что также не считается лучшей практикой. Чтобы решить эту проблему,
 Spring Data также позволяет экстернализировать запросы в файле свойств. Вы можете создать папку с именем META-INF
 внутри папки srcmainresources. Добавьте файл с именем jpa-named-queries.properties в папку META-INF.
  Затем вы можете экстернализировать запросы в формате Entity.finderMethod=Query.
  Например, вы можете экстернализировать запрос для метода findAllByCategory(..), как показано ниже:
  Course.findAllByCategory = select c from Course c, где c.category=?1. Spring Data автоматически обращается
  к этому внешнему запросу, когда ему необходимо выполнить метод findAllByCategory(..)
